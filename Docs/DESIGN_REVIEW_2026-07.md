# 디자인 검수 및 개선 계획 (2026-07)

레퍼런스 목업(보드 편집 화면)과 실제 렌더링 결과의 격차를 진단하고, 경쟁앱 분석을 바탕으로
"공유하고 싶은 결과물"이 되기 위한 개선을 정리한다. P0 항목은 이번 변경에서 코드로 반영했다.

## 1. 보드 이미지 품질 진단 — 왜 조잡해 보였나

| # | 원인 | 증상 | 반영 |
|---|------|------|------|
| 1 | 지도 위 크림색 38% 불투명 플랫 오버레이 + 과도한 desaturate | 지도가 빈 베이지 판처럼 워시아웃, 길·강·라벨 소실 | ✅ 오버레이 제거, `saturation 0.62 → colorMultiply(0xF0E5CA)` 곱셈 톤으로 교체(대비 유지 + 세피아) |
| 2 | 지도 `scaledToFit` | 9:16/4:5 내보내기에서 지도 밖 단색 레터박스 | ✅ `scaledToFill` + clip |
| 3 | 스냅샷 region span ×2.10 | 도시 전체가 잡혀 골목 디테일 소실 | ✅ ×1.45, 최소 delta 축소 (골목 수준 줌) |
| 4 | 코르크 = 그라디언트 + 랜덤 사각 플렉 820개 | 코르크가 아니라 노이즈로 보임 | ✅ 미세 타원 그레인 1,900 + 대형 얼룩 + 비네트 |
| 5 | 실이 교차 방향 베지어 + 두꺼운 검정 언더레이 | S자로 출렁이는 스파게티 | ✅ 핀→핀 사이 중력 새그(catenary 근사) + 오프셋 그림자 + 대시 하이라이트로 꼬임 질감 |
| 6 | 카드 배치가 지리와 무관한 고정 슬롯 | 지도 위 보드인데 동선이 지도와 불일치 | ✅ `normalizedPoints`(실제 좌표) 시드 → 겹침 완화(relaxation) 배치, 퇴화 케이스는 큐레이티드 슬롯 폴백 |
| 7 | 타이틀 부제 "여름의 하루" 하드코딩 | 계절 무관 오류 | ✅ 날짜 기반 계절 문구 |
| 8 | 핀 라스터 에셋 + 무거운 단일 그림자 | 저해상도에서 뭉개짐, 그림자 방향 비일관 | ✅ 벡터(Canvas) 유리알 핀 + 접점 그림자, 시트별 분리 그림자 |

검증 방법: Linux 환경에서 SwiftUI를 실행할 수 없어, 동일한 수치(색·비율·그림자·레이아웃 수식)를
HTML Canvas로 1:1 구현해 3회 반복 스크린샷 비교 후 확정값을 Swift로 포팅했다.
레이아웃 완화 알고리즘은 Python 미러로 16개 케이스(퇴화·수직 동선·8장소 등) 무겹침을 수치 검증했고,
`PhotoClustererTests`에 동일한 검증 테스트를 추가했다.

### 확정 디자인 토큰 (보드 캔버스)

- 코르크: `0xB08556 → 0x89562D` 그라디언트, 비네트 `0x281204 @ 0.42`
- 종이 지도: 바탕 `0xEDE7D8`, 지도 `saturation 0.62 · contrast 0.96 · colorMultiply 0xF0E5CA`
- 폴라로이드: 카드 `0xFDFBF4`, 비율 1:1.24, 사진 폭의 86%, 패딩 5.5%
- 캡션: 장소명 bold 7.4%w, 장수 `0xB03A2E`, 시간대 `0x3A3428 @ 72%`
- 실: 기본 주홍 `0xBE3D2C`, 새그 `min(4.2%w, 길이의 8.5%)`
- 핀 팔레트: 블루/틸/옐로/크림/레드/그린 (라이트·다크 쌍으로 구체 셰이딩)

## 2. 경쟁앱 분석 → 시사점

| 앱 | 강점 | MapRibbon이 취할 것 |
|----|------|---------------------|
| Polarsteps | GPS 자동 트래킹, Trip Reels(지도 플라이오버 영상), 팔로우 공유 | 결과물의 "시네마틱함". 단, 우리는 추적 없이 사진 메타데이터만으로 동일 감성 도달이 차별점 |
| Relive | 액티비티 3D 경로 영상 | 영상화는 MVP 이후. 정적 이미지 품질이 먼저 |
| 포토로그 | 지도에 사진 채우기, 커플 공유 | 지역 수집(아틀라스)과 결합된 리텐션 루프 — 이미 보유, 노출 강화 필요 |
| Tripnote | 방문 국가/지역 수집 지도 | 수집 욕구 = 재방문 동기. 아틀라스 진행률을 홈에 노출할 것 |
| Wanderlog | 계획+기록 통합 | 계획 기능은 비범위 유지(포지셔닝 명확) |
| Journey | 일기+지도 아틀라스 | 텍스트 메모 필드(카드 캡션 커스텀)로 부분 수용 — 반영함 |

**핵심 결론**: 이 카테고리의 유기적 성장 루프는 "공유된 결과물이 곧 광고"다. 경쟁앱 대비
MapRibbon의 우위는 (1) 추적 없이 자동, (2) 인쇄물 감성의 단 한 장. 따라서 보드 한 장의
완성도가 리텐션과 획득을 모두 결정한다 → 이번 개선의 우선순위 근거.

### 기능 갭 로드맵 제안

1. **P1 — 보드 무드 프리셋**: 코르크 외에 리넨/크라프트지 배경, 흑백 필름 등 3종 (템플릿 피커 실질화)
2. **P1 — 아틀라스 훅 강화**: 보드 저장 직후 "서울 3번째 기록 · 17개 지역 중 6개" 토스트로 수집 루프 연결
3. **P2 — 다일 여행 캐러셀**: 연속 날짜 묶음 → 스와이프형 다장 내보내기 (기존 백로그와 일치)
4. **P2 — 지도 플라이오버 영상**: Polarsteps Trip Reels 대응, 정적 이미지 검증 후

## 3. UI/UX 전체 검수

앱 셸(온보딩/홈/보관함/설정)은 "생산성 도구처럼 중립적" 원칙에 잘 맞게 구현되어 있고 접근성
라벨·reduce motion 대응도 양호하다. 발견된 이슈:

### 반영함 (P0)

- 보드 캔버스 전면 개선 (§1)
- 카드 캡션: 지오코딩 부제("서울 · 서울특별시" 중복 정보) 대신 **머문 시간대**를 기본 표기,
  사용자가 설명을 입력하면 그것을 우선 (레퍼런스 목업과 동일)
- 워터마크: 지도 위 맨글자 → 종이 캡슐 배지로 가독성/품위 개선

### 개선 계획 (P1)

- **에디터 하단 바 중복**: '사진 추가'와 '순서 변경'이 모두 PlaceManager를 염 → '순서 변경'은
  목록의 재정렬 모드로 바로 진입시키거나 통합
- **홈 '최근 보드' 헤더의 "보관함" 텍스트**: 정적 텍스트인데 링크처럼 보임 → 실제 탭 전환
  버튼으로 승격 또는 제거
- **ExportSheet 미리보기**: 비율 전환 시 캔버스 재배치 애니메이션 없음 → `.animation` 한 줄로
  전환 품질 개선 가능
- **저장 완료 피드백**: alert 대신 비차단 토스트 + '아틀라스 보기' 액션 (§2 로드맵 2와 연결)

### 개선 계획 (P2)

- 보드 캔버스 텍스트 한국어 하드코딩("N장", 계절 문구) → `Localizable.xcstrings`로 이동 (일본어 출시 전 필수)
- 온보딩 데모 보드(OnboardingBoardPreview)를 새 캔버스 컴포넌트 재사용으로 교체해 첫인상 통일
- 다크 모드에서 보드 캔버스는 의도적으로 라이트 고정 (인쇄물 컨셉) — 에디터 크롬만 다크 대응 확인 필요
