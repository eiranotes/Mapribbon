# 디자인 검수 및 개선 반영 기록 (2026-07)

레퍼런스 목업과 실제 보드 렌더링의 차이를 줄이고, 저장 이후 아틀라스 재방문 루프까지 연결한 변경 기록이다.
초기 HTML Canvas 결과는 시각 프로토타입으로만 취급하며, 최종 판정은 GitHub Actions의 Xcode 빌드·테스트와 시뮬레이터 PNG 검수로 수행한다.

## 이번 PR 반영 범위

### 보드 결과물

- 지도 위 불투명 크림 오버레이 제거, 세피아 곱셈 톤으로 길·강·라벨 대비 유지
- 모든 비율에서 `scaledToFill`을 사용하되, 원본 3:4 지도 좌표를 9:16·4:5 crop 좌표로 함께 변환
- 사용자 방문 순서를 그대로 렌더링하고 실 연결 순서에도 동일하게 적용
- 실제 지도 좌표를 초기 위치로 유지하며 충돌만 완화하고 최대 이동 거리를 제한
- 완전 중복 좌표만 큐레이티드 슬롯으로 폴백
- 카드, 실, 압정, 제목 노트, 워터마크 렌더링 개선
- 코르크·리넨·크라프트 계열 템플릿을 실제 배경·종이·카드 색으로 구분
- 미리보기 해상도에서는 입자 수를 줄이고 고해상도 내보내기에서 세부 밀도를 높이는 적응형 질감

### 편집·저장 흐름

- 중복되던 `사진 추가`/`순서 변경` 진입점을 `장소·사진` 하나로 통합
- 장소의 역지오코딩 주소와 사용자 캡션을 별도 필드로 분리
- 저장 완료 alert를 비차단 토스트로 교체
- 저장된 지역의 기록 횟수와 아틀라스 진행률을 표시하고 `아틀라스 보기`로 탭 이동
- 홈의 동작하지 않는 `보관함` trailing 텍스트 제거
- 온보딩 미리보기가 실제 `BoardCanvasView`를 재사용하도록 통일
- 출력 비율 전환에 Reduce Motion을 존중하는 220ms 애니메이션 추가

### 현지화

- 공유 이미지 내부의 사진 장수와 계절 문구를 한국어·일본어 문자열 카탈로그로 이동
- 저장 토스트와 아틀라스 진행 문구 일본어 추가
- 일본어 로케일 보드 스크린샷을 CI 캡처 항목에 추가

## 검증 기준

- 장소 1~8개 × 9:16·4:5·3:4 전 조합에서 카드 경계 및 겹침 검사
- 지도 좌표 기반 배치의 최대 이동 거리 검사
- 동일 좌표 퇴화 입력 폴백 검사
- `scaledToFill` crop과 정규화 좌표 변환 수치 검사
- 앱 타깃 빌드 후 스크린샷을 먼저 캡처하고, 테스트 타깃 오류가 시각 증거 생성을 막지 않도록 CI 순서 분리
- Xcode 결과 번들, 빌드·테스트 로그, 한국어·일본어 PNG를 Actions artifact로 보존

## 후속 범위

## 2차: 앱 셸 비주얼 아이덴티티 — "측량 도판(Survey Plate)" 시스템 (frontend-design 스킬 기준)

중립 iOS 그레이 셸이 결과물과 단절되어 있고 특색이 없다는 피드백에 따라, 앱 셸에 전용 비주얼 언어를 도입했다.

- **컨셉**: 보드가 코르크 보드 결과물이라면 앱 셸은 그것을 만드는 지도 제작실. 빈티지 측량 도판의 세계.
- **시그니처(대담함은 한 곳)**: 주홍 박음질 실 — 홈의 도판 사이 세로 스티치, 섹션 헤더 밑줄 스티치,
  보드 생성 진행률(다섯 핀 사이를 실이 꿰매는 `MRRouteProgress`), 기본 버튼의 안쪽 박음질 테두리.
- **팔레트**: 잉크 `#22303F` · 해도지 `#EFE9DD`(다크: 야간 해도실 `#131B24`) · 주홍 실 `#C7402D` · 놋쇠 `#A5793C` · 해수 `#52808F`
- **텍스처**: 플랫 배경 대신 서베이 그리드(`MRSurveyGrid`) — 26pt 격자 + 4배 주격자 + 교차 십자.
- **도판 프레임**: 이중 괘선 + 변 눈금(`MRPlateFrame`) + 놋쇠 핀(`MRPinDot`) — 히어로 카드·보드 포스터·온보딩 미리보기·페이월 샘플에 적용.
- **타이포**: 큰 제목 고딕 heavy + 좁은 자간, 날짜·숫자·이어브로는 세리프(New York). 한글 세리프 부재로 세리프 액센트는 라틴·숫자에 한정.
- **AI 디폴트 회피 근거**: 크림+세리프+테라코타 디폴트와 달리 (1) 그리드 텍스처 지면, (2) 도판 괘선 프레임,
  (3) 장식이 아닌 실데이터 메타(도판 날짜·장수), (4) 스티치 시그니처로 차별화.

적용 화면: 홈, 보관함, 온보딩, 보드 생성 진행, 아틀라스 헤더·진행 링, 설정 배경, 페이월.
